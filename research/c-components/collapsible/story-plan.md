# Collapsible — story plan

8 stories. Collapsible's real complexity lives in the Panel's measurement/animation engine (brief.md §6), so the story set weights heavily toward proving each animation-related prop/scenario works and is visually verifiable, on top of the baseline open/close interaction. Several stories recreate scenarios the maintainers themselves manually test in `docs/src/app/(private)/experiments/collapsible/*.tsx` (brief §11) — cite the source experiment in each story's comment where applicable.

1. **Open/close play with `aria-expanded` assertion** — the required baseline interaction story (per the task's instruction). Hero-demo styling (`Collapsible.Root > Collapsible.Trigger + Collapsible.Panel > Content`). Play: `render` → assert `aria-expanded="false"` and the panel absent/hidden → click Trigger → `waitFor` `aria-expanded="true"` → assert panel content is now queryable via `within(canvasElement)` → click again → `waitFor` `aria-expanded="false"`. One assertion per `waitFor` per AGENTS.md. Serves the a11y-contract section directly.
2. **Animated height via `--collapsible-panel-height`** — reproduces the hero-demo CSS recipe verbatim (brief §8): `.Panel { height: var(--collapsible-panel-height); overflow: hidden; transition: height 150ms ease-out; } .Panel[data-starting-style], .Panel[data-ending-style] { height: 0; }`. Play: open the panel and assert the Panel's computed/inline height transitions away from `0` (using `preview_inspect`-equivalent height assertions in the vitest browser mode, or a `getComputedStyle` check inside the play function) — proves the CSS var is actually being written and consumed, not just present in the stylesheet.
3. **`keepMounted`** — Panel with `keepMounted` set, Root `defaultOpen={false}`. Assertion: the Panel's DOM node is queryable (via `container.querySelector` bypassing normal visibility-based queries, since it's present-but-hidden) even while closed, contrasting with the default story's absent-when-closed behavior. Recreates the `transitions.tsx` experiment's `keepMounted: true` vs `keepMounted: false` side-by-side comparison (brief §9, §11) — consider structuring this as two panels side by side in one story, matching the experiment's own layout, for maximum illustrative value.
4. **`hiddenUntilFound`** (browser find-in-page) — Panel with `hiddenUntilFound`. Must be `it.skipIf(isJSDOM)`-equivalent / Chromium-project-only per AGENTS.md testing guidance, since the underlying `beforematch` event and `hidden="until-found"` attribute are browser features the test suite itself gates behind `!('onbeforematch' in window)` (brief §7, `CollapsiblePanel.test.tsx`'s own skip condition). Play: assert the panel has `hidden="until-found"` (not a plain boolean `hidden`) while closed; if practical within Storybook's play-function constraints, dispatch a synthetic `beforematch` event and assert the panel opens as a result — this is the single most distinctive capability Collapsible offers over a hand-rolled show/hide div (brief §2, §8) and deserves the most detailed play of the set.
5. **Controlled** — `open`/`onOpenChange` externally driven by a sibling button outside the Collapsible tree (not the Trigger itself), demonstrating `open` can be toggled from anywhere, and that `onOpenChange`'s `eventDetails.reason` distinguishes `'trigger-press'` from other triggers. Recreates the `transitions.tsx` experiment's `controlled` section (brief §9, §11) and touches on the historical `#4553` bug (controlled + keepMounted interaction, since fixed) — a good place for a code comment noting the regression this pattern once had.
6. **Disabled** — Root `disabled`. Play: click the Trigger and assert `onOpenChange` is never called and `aria-expanded` never changes (mirrors `CollapsibleRoot.test.tsx`'s "does not toggle or call onOpenChange when clicked while disabled"). Also assert the Trigger remains focusable (per the `focusableWhenDisabled` convention, brief §7) despite being inert.
7. **Nested collapsibles** — one `Collapsible.Root` whose Panel's Content contains a second, independent `Collapsible.Root`. Recreates the `transitions.tsx` experiment's "nested" section verbatim (brief §5, §9). Play: open the outer panel, then independently open the inner one; assert both track state independently and that closing the outer one doesn't require the inner one to have been closed first (no forced coordination — the point being made is precisely that Collapsible does NOT coordinate, unlike Accordion).
8. **Content resize while open (dynamic height)** — an open panel whose content height changes after the fact (e.g. a button inside the panel that appends a new line of text), recreating the fixed behavior from "Allow content to resize naturally" (#2043, brief §9). Play: open the panel, capture its height, trigger the content change, and assert the panel's rendered height grows to match — proving the `--collapsible-panel-height` var re-measures rather than freezing at the value captured during the open transition.

**Considered and deferred**: a dedicated CSS-keyframe-animation story (as opposed to the CSS-transition recipe in story 2) — the brief's §6 documents meaningfully different logic for the keyframe path (mount-animation suppression, `React.Activity` guards), but authoring a second full animation story risks diluting the required story count without a correspondingly strong illustrative gain over story 2; flagged as a good candidate if the story budget is later expanded (e.g. during Phase E's real Storybook authoring, if time permits beyond this lean Tier-3 floor).

**MDX docs page notes**: per brief §7, Collapsible's docs today cite no W3C ARIA APG pattern at all — the MDX page should add the Disclosure (Show/Hide) Pattern link (https://www.w3.org/WAI/ARIA/apg/patterns/disclosure/) since none of the current live docs do this, and note explicitly (as this brief's §1 does) that Accordion is built from the same primitive — useful "when not to use" framing that the current docs page also omits entirely (it has no Usage guidelines section at all, unlike Input/Select which do).
