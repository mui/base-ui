# Fieldset — story plan

5 stories. Fieldset is a 2-part, minimal component (Root/Legend) whose entire value is (a) staying a native `<fieldset>` for free disabled-cascade + grouping semantics, and (b) replacing only the unstylable `<legend>` with a styleable `<div>`+`aria-labelledby` (brief.md §2, §7). The story set proves both halves plus every composition shape evidenced in the forms handbook.

1. **Legend + grouped heterogeneous fields** — the component's own hero-demo shape: `Fieldset.Root > Fieldset.Legend + Field.Root + Field.Root` (two unrelated fields, e.g. "Company"/"Tax ID" under a "Billing details" legend). No `render` composition. Assertion: the rendered `<fieldset>` has `aria-labelledby` pointing at the Legend's `<div>` id (`getByRole('group')` — native fieldset's implicit role — `toHaveAccessibleName`). Serves DoD anatomy + hero-equivalent sections.
2. **Radio-group composition** — `Field.Root > Fieldset.Root render={<RadioGroup />} > Fieldset.Legend + Radio.Root×N`, matching the forms handbook's "Storage type" example verbatim (brief §5). Play: assert the rendered element is a `<fieldset>` (not an extra wrapping div), that it carries `aria-labelledby`, and that keyboard arrow-navigation between radios still works (RadioGroup's own behavior, unaffected by the Fieldset wrapping).
3. **Checkbox-group composition with `Field.Item` labeling** — `Field.Root > Fieldset.Root render={<CheckboxGroup />} > Fieldset.Legend + (Field.Item > Checkbox.Root + Field.Label + Field.Description)×N`, matching the handbook's "Backup schedule" example verbatim. Play: assert each checkbox has its own accessible name from its `Field.Item`-scoped `Field.Label` (distinct from the group's own Legend-derived name) — proves the two labeling layers (group vs. item) coexist correctly. This is the story that most directly demonstrates "how it labels grouped fields vs Field" per the task's framing.
4. **Disabled cascade** — two sub-cases in one story (or split into 4a/4b if the story file convention prefers one interaction per story):
   - 4a. Nested Fieldsets: outer `Fieldset.Root disabled` wrapping an inner `Fieldset.Root` with no `disabled` prop set — assert the inner fieldset's rendered `<fieldset>` element is still disabled (native attribute) despite never receiving `disabled` directly (brief §6, §9's #944/#4890 lineage).
   - 4b. Composed-over disabled: `Fieldset.Root disabled render={<RadioGroup />}` — assert the resulting element carries `aria-disabled="true"` (RadioGroup's convention) rather than a plain `disabled` DOM attribute, showing that Fieldset passes `disabled` through without normalizing the target's own convention (brief §6, §10 pitfall 4).
5. **Multi-thumb Slider composition** — `Field.Root > Fieldset.Root render={<Slider.Root />} > Fieldset.Legend + Slider.Control > Slider.Track > Slider.Thumb×2 (each with its own aria-label)`, the handbook's "Price range" example verbatim. Demonstrates the one documented case where Fieldset groups non-repeated-Field.Item controls — each thumb keeps its own `aria-label` (not `Field.Label`) since a Slider thumb isn't a Field-integrated control the way a Radio/Checkbox is.

**Not included, and why**: no standalone "just Fieldset.Root with no composition and no grouped fields" story — a Fieldset around nothing demonstrates nothing beyond what story 1 already covers with real content. No dedicated a11y-contract story isolating `aria-labelledby` alone, since every story above already asserts it as a side effect of its primary point (matches the brief's "kept functionality demos + one story per use case" test rather than adding a redundant single-assertion story).

**MDX docs page notes**: the "does it render native `<fieldset>`/`<legend>`?" question (brief §7) is exactly the kind of thing this component's docs page currently omits — the div-vs-legend rationale (cited from #3044, with the external adrianroselli.com reference) belongs in a "Behavior" or "Accessibility" prose section, not left implicit in the API table. This is the single highest-value piece of prose this Storybook pass can add for Fieldset that the current live docs page (`page.mdx`) does not have at all.
