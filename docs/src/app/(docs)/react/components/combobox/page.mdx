## Usage guidelines

- **Combobox is a filterable Select**: Use Combobox when the input is restricted to a set of predefined selectable items, similar to [Select](/react/components/select) but whose items are filterable using an input. Prefer using Combobox over Select when the number of items is sufficiently large to warrant filtering.
- **Avoid for simple search widgets**: Combobox does not allow free-form text input. For search widgets, consider using [Autocomplete](/react/components/autocomplete) instead.
- **Avoid when not rendering an input**: Use [Select](/react/components/select) instead of Combobox if no input is being rendered, which includes accessibility features specific to a listbox without an input.
- **Form controls must have an accessible name**: If `<Combobox.Input>` is the form control, label it with a native `<label>` or `<Field.Label>`, or provide an `aria-label` when no visible label is rendered. `<Combobox.Label>` labels `<Combobox.Trigger>` and is intended for the [input-inside-popup](#input-inside-popup) pattern, where the trigger is the form control. See the [forms guide](/react/handbook/forms).

## Anatomy

Import the components and place them together:

```jsx title="Anatomy"
import { Combobox } from '@base-ui/react/combobox';

<Combobox.Root>
  <Combobox.Label />

  <Combobox.InputGroup>
    <Combobox.Input />
    <Combobox.Trigger />
    <Combobox.Icon />
    <Combobox.Clear />
    <Combobox.Value />

    <Combobox.Chips>
      <Combobox.Chip>
        <Combobox.ChipRemove />
      </Combobox.Chip>
    </Combobox.Chips>
  </Combobox.InputGroup>

  <Combobox.Portal>
    <Combobox.Backdrop />
    <Combobox.Positioner>
      <Combobox.Popup>
        <Combobox.Arrow />

        <Combobox.Status />
        <Combobox.Empty />

        <Combobox.List>
          <Combobox.Row>
            <Combobox.Item>
              <Combobox.ItemIndicator />
            </Combobox.Item>
          </Combobox.Row>

          <Combobox.Separator />

          <Combobox.Group>
            <Combobox.GroupLabel />
          </Combobox.Group>

          <Combobox.Collection />
        </Combobox.List>
      </Combobox.Popup>
    </Combobox.Positioner>
  </Combobox.Portal>
</Combobox.Root>;
```

## Item values

When an item is rendered by the function child of `<Combobox.List>` or `<Combobox.Collection>`, its source item is automatically used as its value, so the `value` prop on `<Combobox.Item>` can be omitted. Items rendered manually without a collection require an explicit `value` prop.

To use a primitive value that differs from the source item—such as an ID held in an object—normalize the `items` array with the `Combobox.useItems` hook, which derives each item's value and label before rendering.

## Examples

### Typed wrapper component

The following example shows a typed wrapper around the Combobox component with correct type inference and type safety:

```tsx title="Specifying generic type parameters"
import * as React from 'react';
import { Combobox } from '@base-ui/react/combobox';

export function MyCombobox<Value, Multiple extends boolean | undefined = false>(
  props: Combobox.Root.Props<Value, Multiple>,
): React.JSX.Element {
  return <Combobox.Root {...props}>{/* ... */}</Combobox.Root>;
}
```

### Value selection with IDs

When your app stores values as IDs rather than whole item objects, normalize the `items` array with the `Combobox.useItems` hook. The `value` accessor derives the primitive value that identifies each item, and the `label` accessor derives the string shown in the input, which is also used for filtering and typeahead.

Each item's derived value is used as its selection value automatically, so `defaultValue`, `value`, and `onValueChange` all work with plain IDs. The label is resolved from the collection even while the items are unmounted, so the input can display the selected item's label when the popup is closed.

import { DemoComboboxUseItems } from './demos/use-items';

<DemoComboboxUseItems />

In React Server Components, use the hook-free `Combobox.items()` variant. It applies the accessors eagerly on the server and returns a serializable result. Pass that result through `useItems()` in the client component that renders the combobox:

```tsx title="Normalizing items on the server"
// page.tsx (server)
const serverItems = Combobox.items(fruits, {
  value: (fruit) => fruit.id,
  label: (fruit) => fruit.label,
});
// Pass `serverItems` as a prop to a client component.

// picker.tsx ('use client')
const items = Combobox.useItems(props.serverItems);
```

### Multiple select

The combobox can allow multiple selections by adding the `multiple` prop to `<Combobox.Root>`.
Selection chips are rendered with `<Combobox.Chip>` inside the input that can be removed.

import { DemoComboboxMultiple } from './demos/multiple';

<DemoComboboxMultiple />

Visible chips can be limited by slicing the selected values rendered in `<Combobox.Value>`:

```tsx title="Limiting visible chips"
const CHIP_LIMIT = 3;

<Combobox.Value>
  {(selectedValue: string[]) => {
    // @highlight-start
    const visibleValue = selectedValue.slice(0, CHIP_LIMIT);
    const hiddenCount = selectedValue.length - visibleValue.length;
    // @highlight-end
    return (
      <>
        {visibleValue.map((item) => (
          <Combobox.Chip key={item}>
            {item}
            <Combobox.ChipRemove aria-label={`Remove ${item}`} />
          </Combobox.Chip>
        ))}
        {/* @highlight-start */}
        {hiddenCount > 0 && <span>{`+${hiddenCount} more`}</span>}
        {/* @highlight-end */}
        <Combobox.Input />
      </>
    );
  }}
</Combobox.Value>;
```

Use `<Combobox.Label>` to provide a visible label for the combobox trigger in this pattern:

```tsx title="Using Combobox.Label to label a combobox"
<Combobox.Root>
  {/* @highlight */}
  <Combobox.Label>Favorite fruit</Combobox.Label>
  {/* ... */}
</Combobox.Root>
```

`<Combobox.Label>` renders a `<div>`, so clicking it focuses the combobox trigger without opening the popup.

### Grouped

Organize related options with `<Combobox.Group>` and `<Combobox.GroupLabel>` to add section headings inside the popup.

Groups are represented by an array of objects with an `items` property, which itself is an array of individual items for each group. An extra property, such as `value`, can be provided for the heading text when rendering the group label.

```tsx title="Example"
interface ProduceGroupItem {
  value: string;
  // @highlight
  items: string[];
}

const groups: ProduceGroupItem[] = [
  {
    value: 'Fruits',
    // @highlight
    items: ['Apple', 'Banana', 'Orange'],
  },
  {
    value: 'Vegetables',
    // @highlight
    items: ['Carrot', 'Lettuce', 'Spinach'],
  },
];
```

#### Memoizing items

Memoizing each item is a simpler alternative to virtualization for datasets up to roughly 1,000 items. Wrap each item in [`React.memo`](https://react.dev/reference/react/memo) and pass the item as a prop so unchanged items skip re-rendering. While memoization speeds up typing, it does not speed up opening; with a large enough number of items, the mount cost dominates, and virtualization becomes necessary to keep the open interaction fast on low-end devices.

```tsx title="Memoizing list items"
interface Fruit {
  id: string;
  label: string;
}

const FruitItem = React.memo(function FruitItem({ item }: { item: Fruit }) {
  return (
    <Combobox.Item>
      <Combobox.ItemIndicator />
      <span>{item.label}</span>
    </Combobox.Item>
  );
});

<Combobox.List>{(item: Fruit) => <FruitItem key={item.id} item={item} />}</Combobox.List>;
```

## useFilter

Matches items against a query using `Intl.Collator` for robust string matching.
This hook is used when externally filtering items.
Pass the result to the `filter` prop of `<Combobox.Root>`.
