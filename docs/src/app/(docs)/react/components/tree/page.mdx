## Anatomy

Import the component and assemble its parts:

```jsx title="Anatomy"
import { Tree } from '@base-ui/react/tree';

<Tree.Root>
  <Tree.Item>
    <Tree.ItemExpansionTrigger />
    <Tree.ItemGroupIndicator />
    <Tree.ItemLabel />
    <Tree.ItemLoadingIndicator />
    <Tree.ItemErrorIndicator />
  </Tree.Item>
  <Tree.CheckboxItem>
    <Tree.ItemExpansionTrigger />
    <Tree.CheckboxItemIndicator />
    <Tree.ItemGroupIndicator />
    <Tree.ItemLabel />
    <Tree.ItemLoadingIndicator />
    <Tree.ItemErrorIndicator />
  </Tree.CheckboxItem>
</Tree.Root>;
```

For animated expand/collapse, wrap the item render function in `Tree.AnimatedItemList`:

```jsx title="Animated anatomy"
<Tree.Root>
  <Tree.AnimatedItemList>
    {(item, animatedChildren) => (
      <React.Fragment>
        <Tree.Item>
          <Tree.ItemExpansionTrigger />
          <Tree.ItemLabel />
        </Tree.Item>
        <Tree.GroupTransition>{animatedChildren}</Tree.GroupTransition>
      </React.Fragment>
    )}
  </Tree.AnimatedItemList>
</Tree.Root>
```

## Examples

### Checkbox selection propagation

Use `checkboxSelectionPropagation` to control how selection changes propagate through the tree hierarchy when using `Tree.CheckboxItem`. By default, both `parents` and `descendants` are enabled: selecting a parent selects all its children, and a parent becomes selected when all its children are selected. Toggle the checkboxes below to see how each option affects selection behavior.

import { DemoTreeSelectionPropagation } from './demos/selection-propagation';

<DemoTreeSelectionPropagation compact />

### Checkbox items

`<Tree.Item>` uses **replace** selection: clicking an item selects it and deselects others (use Ctrl/Cmd to add, Shift to range-select).

`<Tree.CheckboxItem>` uses **toggle** selection: clicking an item toggles its selection without affecting others. It also renders checkbox ARIA semantics (`aria-checked` instead of `aria-selected`) and supports indeterminate state for parent items with partially selected children.

import { DemoTreeCheckboxItems } from './demos/checkbox-items';

<DemoTreeCheckboxItems compact />

### Animated expand/collapse

Enable smooth height transitions when expanding and collapsing tree items using `Tree.AnimatedItemList` and `Tree.GroupTransition`. The render function receives `animatedChildren` as its second argument — pass it as children of `Tree.GroupTransition` to enable animated transitions. The wrapper exposes `--tree-group-height`, `data-starting-style`, and `data-ending-style` for CSS-based animations.

import { DemoTreeAnimated } from './demos/animated';

<DemoTreeAnimated compact />

### Lazy loading

Load tree items on demand when a parent node is expanded. This is useful for large data sets where loading the entire tree upfront would be expensive.

import { DemoTreeLazyLoading } from './demos/lazy-loading';

<DemoTreeLazyLoading compact />

### Filtering

Filter tree items by text query using `Tree.useFilter()` and `Tree.useFilteredItems()`. Matching items are shown along with their ancestors to preserve the tree structure.

import { DemoTreeFiltering } from './demos/filtering';

<DemoTreeFiltering compact />

### Virtualized

Render large trees efficiently by only rendering items currently visible in the viewport. Set `virtualized` on `Tree.Root` and use `Tree.useVisibleItems()` to get the flat list of visible items, then wire up your own virtualizer (e.g. `@tanstack/react-virtual`). Each `Tree.Item` must receive an `itemId` prop.

## useVisibleItems

Returns the flat list of visible items (respecting the current expansion state) when called inside `<Tree.Root>`. Use this hook with `virtualized` to get the items to render.
