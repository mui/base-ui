## Usage guidelines

- **Form controls must have an accessible name**: It can be created using a `<label>` element or the `Field` component. See [Labeling an OTP field](#labeling-an-otp-field) and the [forms guide](/react/handbook/forms).

## Anatomy

Import the component and assemble its parts:

```jsx title="Anatomy"
import { OTPField } from '@base-ui/react/otp-field';

<OTPField.Root length={6}>
  <OTPField.Group>
    <OTPField.Input />
  </OTPField.Group>
</OTPField.Root>;
```

## Examples

### Labeling an OTP field

Use a native `<label>` with the root `id` to label the OTP field, and place `aria-describedby` on
`<OTPField.Root>` when you need supporting description text:

```tsx title="OTP Field with a native label and description" {2,4,14}
<OTPField.Root id="verification-code" length={6} aria-describedby="verification-code-description">
  <label htmlFor="verification-code">Verification code</label>
  <OTPField.Group>
    <OTPField.Input aria-label="Character 1 of 6" />
    <OTPField.Input aria-label="Character 2 of 6" />
    <OTPField.Input aria-label="Character 3 of 6" />
    <OTPField.Input aria-label="Character 4 of 6" />
    <OTPField.Input aria-label="Character 5 of 6" />
    <OTPField.Input aria-label="Character 6 of 6" />
  </OTPField.Group>
  <p id="verification-code-description">Enter the 6-character code we sent to your device.</p>
</OTPField.Root>
```

### Form integration

Use [Field](/react/components/field) to handle label associations and form integration:

```tsx title="Using OTP Field in a form" {2}
<Form>
  <Field.Root name="verificationCode">
    <Field.Label>Verification code</Field.Label>
    <Field.Description>Enter the 6-character code we sent to your device.</Field.Description>
    <OTPField.Root length={6}>
      <OTPField.Group>
        <OTPField.Input aria-label="Character 1 of 6" />
        <OTPField.Input aria-label="Character 2 of 6" />
        <OTPField.Input aria-label="Character 3 of 6" />
        <OTPField.Input aria-label="Character 4 of 6" />
        <OTPField.Input aria-label="Character 5 of 6" />
        <OTPField.Input aria-label="Character 6 of 6" />
      </OTPField.Group>
    </OTPField.Root>
  </Field.Root>
</Form>
```

Pass `autoSubmit` to submit the owning form automatically when all slots are filled, or use
`onValueComplete` to react to completion without submitting.

### Alphanumeric verification codes

Use `validationType="alphanumeric"` for recovery, backup, or invite codes that mix letters and
numbers.

<DemoOTPFieldAlphanumeric compact />

### Masked entry

Use `mask` when the code should be obscured while it is being typed. If you need a custom input
type, pass `type` directly to individual `<OTPField.Input>` parts.

<DemoOTPFieldPassword compact />

### Custom sanitization

Set `validationType="none"` with `sanitizeValue` when you need to normalize pasted values before
they reach state, such as removing spaces or hyphens and uppercasing invite codes.
